# GigaAM-v3 transcription engine вАФ Design

## Problem

Add Sber's GigaAM-v3 (best-in-class Russian ASR) as a third local transcription
engine in Meetily, alongside Whisper (whisper.cpp) and Parakeet (ONNX). Meetily
users recording Russian meetings currently rely on Whisper/Parakeet, which are
weaker on Russian than GigaAM-v3.

## Feasibility (confirmed by research)

- GigaAM-v3 ships as ready ONNX at HF `istupakov/gigaam-v3-onnx` (MIT). Same
  author as the Parakeet ONNX already used by Meetily вЖТ packaging is analogous.
- Input: 16 kHz mono float вАФ identical to Meetily's pipeline and Parakeet.
- `ort` (ONNX Runtime) is already a dependency (Parakeet) вЖТ no new native dep.
- Variant chosen: **e2e-ctc** вАФ single encoder ONNX (`v3_e2e_ctc[.int8].onnx`) +
  `v3_e2e_ctc_vocab.txt`. "e2e" adds punctuation + text normalization. CTC greedy
  decode (argmax вЖТ collapse repeats вЖТ drop blank) is simple and robust to
  implement without local runtime iteration.
- Encoder I/O (from onnx-asr reference): inputs `features [1,64,T]` +
  `feature_lengths [1]`, output `log_probs [1,T',V]`; output length =
  (feature_len-1)//4 + 1 (subsampling_factor 4).
- Feature extraction: log-mel, 64 mels, n_fft=320, win=320, hop=160, htk mel
  scale, center=false, no mel_norm. GigaAM has NO preprocessor ONNX shipped
  (unlike Parakeet's nemo128.onnx).

## Decision: preprocessor as ONNX (mirror Parakeet)

Rather than reimplement log-mel in Rust, generate a **preprocessor ONNX**
(waveform вЖТ features) from the model's documented params using onnx-asr's
builder (pure Python, no crates.io). Host it next to the model. The Rust engine
then mirrors `parakeet_engine/model.rs`: preprocessor.onnx (audioвЖТfeatures) вЖТ
encoder.onnx (featuresвЖТlog_probs) вЖТ CTC decode. This reuses the reference
feature computation (accuracy-safe) and matches Meetily's existing code path.

Fallback if the preprocessor ONNX proves impractical: implement log-mel in Rust
with the existing `realfft` dep from the exact params above.

## De-risking: Python reference first

Before writing Rust: locally (Python) generate the preprocessor ONNX and run the
full e2e-ctc pipeline on a sample Russian wav to confirm the ONNX files + I/O +
feature params produce a correct Russian transcript. Only then port to Rust.

## Components (mirror Parakeet pattern)

1. `src-tauri/src/gigaam_engine/{mod,gigaam_engine,model,commands}.rs` вАФ engine,
   ONNX model wrapper, Tauri commands (init, get_available_models, download,
   load, transcribe, validate, delete). Model dir: `app_data/models/gigaam/<name>/`.
2. `config.rs` вАФ GIGAAM model registry (name, files, size, description) +
   `DEFAULT_GIGAAM_MODEL`. `lib.rs` вАФ register module + commands.
3. `audio/transcription/engine.rs` + `api/api.rs` вАФ add `"gigaam"` provider
   branch (validate ready, init engine).
4. Frontend: `TranscriptSettings.tsx` вАФ add `gigaam` provider option
   (рЯЗЈрЯЗЇ GigaAM вАФ –†—Г—Б—Б–Ї–Є–є); `GigaAMModelManager.tsx` вАФ copy of ParakeetModelManager,
   downloads `v3_e2e_ctc.int8.onnx` + `v3_e2e_ctc_vocab.txt` (+ preprocessor onnx)
   from the HF repo / fork release asset.

## Model download

Source files from HF `istupakov/gigaam-v3-onnx` (public): `v3_e2e_ctc.int8.onnx`
(~250-300 MB), `v3_e2e_ctc_vocab.txt`. Preprocessor ONNX generated by us and
attached to a fork release asset (or hosted on HF under our account). Download
lifecycle, progress events, resume, and validation mirror Parakeet.

## Constraints / caveats

- GigaAM-v3 is **Russian-only** (Cyrillic vocab). It is an ADDITIONAL provider,
  not a replacement вАФ Whisper/Parakeet remain for English/multilingual.
- Cannot run the app locally (crates.io blocked) вЖТ build on fork CI, user tests
  Russian audio manually. Python reference validates the model pipeline first.
- Ships as version bump (0.4.2) so the existing auto-update delivers it.

## Testing

- Python reference: correct Russian transcript on a sample wav.
- `tsc --noEmit`; fork CI build success (Rust compiles with new engine).
- Manual: select GigaAM in Settings вЖТ Transcription, download model, record a
  Russian phrase, verify accurate punctuated transcript.

## Out of scope (later)
